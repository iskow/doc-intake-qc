  - **Joel answered both open questions.** (1) Preserve everything — ACLs, owner, alternate data streams, not just timestamps. That disqualifies ctypes (timestamps only) and selects robocopy. (2) A filename collision must **fail loudly** with a warning, never auto-suffix `__2`.

Batches capped at 8,000 chars against the Windows command-line limit. Exit codes checked (0–7 = success) because robocopy fails silently otherwise. `copy2` demoted to a loud non-Windows fallback.

Note it can **never fire on real input** — preserving relative paths makes collisions structurally impossible — so the gate proves it fires on a synthetic plan *and* stays silent on the real one.

**Measured copy-method facts, so nobody re-derives them:** `robocopy /COPY:DATSO /DCOPY:DAT` preserves created, modified, accessed, ACL (SDDL identical), owner and alternate data streams, unelevated, exit 1 = success; **`/COPYALL` fails unelevated with exit 16** (needs the audit privilege) and copies nothing; PowerShell `Copy-Item` and Python `copy2` both lose creation time and ACLs, but `copy2` **does** carry alternate data streams (via Win32 `CopyFile2`).

- **Claim what was proven:** all 41 copies' **created and modified** dates match their sources exactly, in all three modes. The seeded 1980 and 2031 anomalies survive the copy.

Say so plainly, and say what a client should do instead: capture access times with the forensic collection tool at acquisition.
